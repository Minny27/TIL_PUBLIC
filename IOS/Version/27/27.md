> - [Xcode 27 Beta Release Notes](https://developer.apple.com/documentation/xcode-release-notes/xcode-27-release-notes)

> 기준: iOS & iPadOS 27 beta, Xcode 27 beta

## Overview

iOS & iPadOS 27 SDK는 iOS 27, iPadOS 27 beta 앱 개발을 지원하며 Xcode 27에 포함된다.

## UIKit

### New Features

#### UIScene state restoration

`UIScene.extendStateRestoration`과 `UIScene.completeStateRestoration`을 사용할 수 있다.

`UIScene.ActivationState.background`에서 `UIScene.ActivationState.foreground`로 전환되는 lifecycle 동안 state restoration을 확장하기 위한 API다.

#### Launch screen 필수

iOS/iPadOS 27 SDK 이상으로 빌드한 앱은 launch screen을 포함해야 한다.

`Info.plist`에는 다음 key 중 하나가 필요하다.

- `UILaunchStoryboardName`
- `UILaunchStoryboards`
- `UILaunchScreen`
- `UILaunchScreens`

앱에 launch screen이 없으면 App Store에서 iOS 27 SDK 빌드 앱을 받기 시작한 이후 reject될 수 있다.

#### UIDragInteraction과 Siri

iOS/iPadOS 27에서 Siri는 앱 UI에 설치된 drag interaction의 resource를 로드할 수 있다.

예를 들어 Apple Intelligence가 context menu에서 실행되면 시스템이 `UIDragInteractionDelegate` method를 호출해 content를 로드할 수 있다.

drag session이 실제 사용자 drag gesture 없이 시작될 수 있으므로 `dragInteraction(_:sessionWillBegin:)`에서 animation이나 modal UI 표시를 피해야 한다.

사용자 drag 중에 필요한 UI 처리는 `dragInteraction(_:sessionDidMove:)`에서 수행하는 것이 권장된다.

#### Presented view controller trait inheritance 변경

iOS 27 SDK로 빌드한 앱에서 presented view controller는 presentation controller로 바로 이동하지 않고, presentation 중간 view들의 superview chain을 따라 trait collection을 상속한다.

custom `UIPresentationController` 또는 presentation controller에서 직접 trait을 받는 구조에 의존하는 view controller는 `traitCollection` 전파나 override 방식을 점검해야 한다.

#### Menu image visibility

iPadOS 27과 macOS 27에서 menu bar와 context menu는 menu item image를 줄여서 표시한다.

`UIMenuElement`에 `preferredImageVisibility` property가 추가되었고, 다음 타입의 initializer도 관련 업데이트를 받았다.

- `UIMenu`
- `UIAction`
- `UICommand`
- `UIKeyCommand`

앱에서 반드시 보여야 하는 menu image는 `preferredImageVisibility`로 명시해야 한다.

#### UISearchController scope bar inline 표시

iOS 27 SDK로 빌드한 앱에서 `UISearchController`가 center search-bar placement를 사용하면 scope bar가 검색 필드 아래 별도 row가 아니라 같은 row에 inline으로 표시된다.

navigation bar 안에 검색 필드가 있으면 scope bar도 navigation bar 안에서 검색 필드 옆에 배치된다.

#### External display non-interactive scene

iOS 27 SDK로 빌드한 앱에서 `windowExternalDisplayNonInteractive` scene은 시스템이 자동으로 제공하지 않는다.

외부 디스플레이에 비상호작용 content를 표시하려면 다음 조합을 사용한다.

- `UIViewController.registerSceneAccessory(_:)`
- `UISceneAccessory.externalNonInteractive`

### Resolved Issues

`UIMenuLeaf` protocol에 iOS 16.0에서 도입된 subtitle property가 누락되어 있던 문제가 수정되었다.

### Known Issues

#### Deprecated status bar accessor

iOS 27 SDK로 빌드한 앱에서 `UIApplication`의 deprecated status bar accessor가 `NaN` 또는 `null` 값을 반환할 수 있다.

영향을 받는 API는 다음과 같다.

- `statusBarFrame`
- `statusBarOrientation`
- `statusBarStyle`
- `isStatusBarHidden`

현재 상태값은 `UIWindowScene.statusBarManager`에서 읽는 것이 권장된다.

#### iPad continuous resize와 orientation

iOS 27 SDK로 빌드한 iPad 앱에서 `UISupportedInterfaceOrientations`가 네 방향을 모두 포함하지 않으면 앱이 continuously resizable로 처리되지 않을 수 있다.

workaround는 `Info.plist`의 `UISupportedInterfaceOrientations`에 네 방향을 모두 선언하는 것이다.

#### UISceneClosureConfirmation

`UISceneClosureConfirmation` API가 confirmation dialog를 표시하지 않는 문제가 있다.

#### Bottom toolbar background

`UIBarButtonItem.hidesSharedBackground`로 toolbar item의 glass background를 숨길 때 bottom toolbar background가 보이지 않을 수 있다.

#### iPhone Mirroring / iPad resize 관련 이슈

iOS 27 SDK 빌드 앱에서 iPhone Mirroring 및 iPad resize 환경의 orientation, `UIRequiresFullScreen`, `UIScreen.main`, scene resize update 관련 known issue가 다수 존재한다.

orientation과 resize에 민감한 layout은 iPad와 iPhone Mirroring에서 별도 검증이 필요하다.

### Deprecations

최신 SDK로 빌드한 앱은 scene-based lifecycle을 채택해야 한다.

scene-based lifecycle을 채택하지 않으면 앱이 launch되지 않을 수 있다.

## TextKit

`NSTextTable`과 관련 object/type이 OS 27부터 UIKit client에서 사용 가능하다.

## StoreKit Testing in Xcode

### Resolved Issues

- `SKTestSession`으로 subscription expiration을 강제한 뒤 unified app receipt가 갱신되지 않던 문제가 수정되었다.
- `SKTestSession.disableDialogs` 설정이 일부 system dialog에서 항상 적용되지 않던 문제가 수정되었다.
- Xcode Transaction Manager에서 수행한 subscription upgrade가 `Transaction.updates`에 보고되지 않던 문제가 수정되었다.
- `SKTestSession`에서 `purchaseDate(_:renewalBehavior:)` purchase option 사용 시 renewal behavior preference가 적용되지 않던 문제가 수정되었다.

### Known Issues

StoreKit Testing in Xcode에서 월간 subscription이 12개월 commitment로 billing되는 경우 `pricingTerms.commitmentInfo.price`가 잘못된 price를 반환할 수 있다.
