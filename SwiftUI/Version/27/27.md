## Apple Documentation Updates

Apple Developer Documentation의 `SwiftUI updates` 중 **June 2026** 항목 기준 정리.

### 핵심 요약

- `@State`가 `State()` macro 기반으로 동작한다.
- 여러 전용 builder를 대체하는 `ContentBuilder`가 추가되었다.
- 리스트, 스택, 그리드, 커스텀 레이아웃에서 drag-and-drop reordering을 더 직접적으로 지원한다.
- 스크롤 뷰, 스택, 그리드, 커스텀 레이아웃에서도 swipe action을 구성할 수 있다.
- `AsyncImage`가 `URLRequest`와 local cache 구성을 지원한다.
- toolbar overflow, visibility priority, minimize behavior 제어가 강화되었다.
- URL 기반 document app을 위한 읽기/쓰기 프로토콜이 추가되었다.
- gesture 입력 source를 직접 지정할 수 있다.

### General

#### `ContentBuilder`

`ContentBuilder`는 타입에 종속되지 않는 content closure를 만들기 위한 builder다.

기존의 전용 builder였던 `ToolbarContentBuilder`, `CommandsBuilder` 같은 타입별 builder를 통합하는 역할을 한다.

#### Reordering

drag-and-drop 기반 reordering을 다음 API로 구성할 수 있다.

- `DynamicViewContent.reorderable()`
- `View.reorderContainer(for:isEnabled:move:)`

적용 대상은 `List`뿐 아니라 stack, grid, custom layout 같은 container까지 확장된다.

#### Swipe Actions

container 내부 view에 custom swipe action을 추가할 수 있다.

- `View.swipeActions(edge:allowsFullSwipe:content:onPresentationChanged:)`
- `View.swipeActionsContainer()`

기존 `List` 중심의 swipe action 패턴을 scroll view, stack, grid, custom layout에서도 사용할 수 있게 하는 변경이다.

### Transitions

`NavigationTransition.crossFade`를 지정하면 sheet가 content 위에 fade-in 형태로 나타나도록 만들 수 있다.

```swift
.navigationTransition(.crossFade)
```

### Toolbars

#### `visibilityPriority(_:)`

toolbar item에 우선순위를 지정해, 공간이 부족할 때 어떤 action을 먼저 overflow menu로 보낼지 제어할 수 있다.

인자는 `ToolbarItemVisibilityPriority` 타입이다.

#### `ToolbarOverflowMenu`

보조 action을 명시적으로 overflow menu에 넣을 수 있다.

archive, delete처럼 항상 노출될 필요는 없지만 접근 가능해야 하는 action에 적합하다.

#### `topBarPinnedTrailing`

toolbar item을 top bar의 trailing edge에 고정할 수 있다.

다른 toolbar item이 overflow menu로 이동하거나 위치가 바뀌어도, 중요한 trailing action을 안정적으로 유지하는 용도다.

#### `toolbarMinimizeBehavior(_:for:)`

scroll 동작에 따라 toolbar가 최소화되는 방식을 제어한다.

### Tab bars

`TabRole.prominent`를 지정하면 tab을 tab bar의 별도 trailing 위치에 배치할 수 있다.

중요도가 높은 tab이나 primary action처럼 보여야 하는 tab에 사용할 수 있다.

### Alerts and Confirmation Dialogs

optional data item이나 error object를 기반으로 alert, confirmation dialog를 표시할 수 있다.

관련 API는 다음과 같다.

- `alert(_:item:actions:)`
- `alert(error:actions:)`
- `alert(_:item:actions:message:)`
- `alert(error:actions:message:)`
- `confirmationDialog(_:item:titleVisibility:actions:)`
- `confirmationDialog(_:item:titleVisibility:actions:message:)`

### Gestures

gesture가 인식할 입력 source를 직접 지정할 수 있다.

예를 들어 direct touch, indirect touch, Apple Pencil, pointer 입력을 구분해서 gesture를 구성할 수 있다.

관련 gesture는 다음과 같다.

- `DragGesture`
- `LongPressGesture`
- `MagnifyGesture`
- `RotateGesture`
- `RotateGesture3D`
- `SpatialEventGesture`
- `SpatialTapGesture`
- `TapGesture`
- `WindowDragGesture`

입력 source 종류는 `GestureInputKinds`를 참고한다.

## New Features

### Status bar color scheme

`toolbarColorScheme(_:for:)`가 status bar를 대상으로 사용할 수 있다.

```swift
.toolbarColorScheme(colorScheme, for: .statusBar)
```

상태바에 선호 color scheme을 지정할 때 사용한다.

### Status bar visibility

`toolbarVisibility(_:for:)`로 status bar 표시 상태를 제어할 수 있다.

```swift
.toolbarVisibility(hideStatusBar ? .hidden : .automatic, for: .statusBar)
```

조건에 따라 status bar를 숨기거나 자동 표시 상태로 되돌릴 수 있다.

### AsyncImage HTTP caching

`AsyncImage`가 HTTP caching protocol을 사용해 downloaded image를 자동 cache한다.

server의 standard HTTP cache header로 cache 동작을 제어할 수 있다.

새 API는 다음 방향으로 추가되었다.

- `URLRequest`를 받는 `AsyncImage` initializer
- custom `cachePolicy` 설정
- child `AsyncImage` data task를 제어하는 `View.asyncImageURLSession(_:)`

```swift
ImageListView()
    .asyncImageURLSession(customSession)
```

### Text selection

iOS/iPadOS 27 SDK로 빌드한 앱에서 `.textSelection(.enabled)`가 적용된 `Text`는 system text selection UI로 user-interactive selection을 지원한다.

기존 iOS/iPadOS selectable `Text`는 callout menu 기반 selection에 가까웠지만, 27 SDK 빌드에서는 system text selection gesture가 추가될 수 있다.

custom gesture가 system text selection보다 우선해야 한다면 `.highPriorityGesture()` 사용을 고려한다.

```swift
Text(content)
    .textSelection(.enabled)
```

### Selectable Text와 TextRenderer

iOS 27, macOS 27 SDK로 빌드한 앱에서 selectable `Text`가 `TextRenderer`를 지원한다.

`.textSelection(.enabled)`와 `.textRenderer(_:)`를 함께 적용했을 때 custom renderer가 무시되던 문제도 수정되었다.

### @State macro 기반 구현

Xcode 27은 macro 기반의 새로운 `@State` 구현을 도입한다.

기존에는 다음처럼 선언한 경우 view struct가 다시 만들어질 때마다 initial expression이 평가될 수 있었다.

```swift
@State private var model = Model()
```

새 구현은 이 반복 평가를 피한다.

이 변경은 iOS 17 aligned OS까지 back deploy된다.

#### 주의할 점

선언부에 초기값을 주고 initializer에서도 값을 대입하는 패턴은 피해야 한다.

```swift
struct ContentView: View {
    @State private var count = 0

    init(count: Int) {
        self.count = count
    }
}
```

초기값을 initializer에서 넣고 싶다면 선언부 초기값을 제거한다.

```swift
struct ContentView: View {
    @State private var count: Int

    init(count: Int) {
        self.count = count
    }
}
```

그 외 주의점은 다음과 같다.

- 모든 stored member가 `private`인 struct에서 compiler가 합성하던 private initializer에 의존하는 코드가 compile되지 않을 수 있다.
- generic argument inference가 드물게 기존보다 덜 유연할 수 있으므로 type을 더 구체적으로 작성해야 할 수 있다.
- `@State`를 다른 property wrapper 또는 macro와 조합하는 것은 지원되지 않는다.

### ReadableDocument / WritableDocument

27 SDK로 빌드한 앱에서 새 document protocol이 추가되었다.

- `ReadableDocument`
- `WritableDocument`

지원하는 기능은 다음과 같다.

- asynchronous reading
- asynchronous writing
- progress reporting
- document URL 직접 접근
- editing-only 앱에서 document creation 비활성화
- document가 열리기 전 custom UI 표시
- `Observable` `URLDocumentConfiguration`
- Swift concurrency 및 Observation framework integration

새 앱은 기존 `ReferenceFileDocument`보다 `ReadableDocument`, `WritableDocument` 사용이 권장된다.

### TabView selection 검증

iOS/iPadOS 27 SDK로 빌드한 앱에서 `TabView`는 selection이 visible tab을 가리키도록 강제한다.

selection이 hidden tab 또는 사용 불가능한 tab을 가리키면 crash가 발생할 수 있다.

`TabView(selection:)`을 사용하는 경우 selection 값과 실제 표시되는 tab 목록의 동기화를 점검해야 한다.

### Menu item image 표시 정책 변경

iPadOS 27, macOS 27의 menu bar와 macOS 27 context menu는 menu item image를 줄여서 표시한다.

SwiftUI는 기본적으로 대부분의 context에서 symbol image를 숨긴다.

menu item image를 항상 보여야 하는 경우 `Label`에 `.labelStyle(.titleAndIcon)`을 적용한다.

```swift
Button {
    action()
} label: {
    Label("Item", systemImage: "star")
}
.labelStyle(.titleAndIcon)
```

### TabsPickerStyle

tab 기반 navigation이나 content selection을 나타내는 picker를 위해 `TabsPickerStyle`이 추가되었다.

`.segmented`와 비슷하지만 VoiceOver가 "tabs"로 읽고, macOS에서는 value selection용 picker와 구분되는 visual appearance를 가진다.

### External non-interactive display accessory

iOS 27 SDK로 빌드한 앱에서 외부 디스플레이 scene에 non-interactive content를 표시할 수 있다.

```swift
.sceneAccessory(...)
```

`ExternalNonInteractiveAccessory` type과 함께 사용한다.

### LabeledContent in Menu

27 SDK로 빌드한 앱에서 `Menu` 내부의 `LabeledContent`는 value를 platform menu item의 subtitle로 mapping한다.

## Resolved Issues

- status bar tap으로 `ScrollView`를 top으로 이동할 때 bound `scrollPosition`이 stale value로 남던 문제가 수정되었다.
- `.textSelection(.enabled)`가 적용된 `Text`에서 `.textRenderer(_:)` custom renderer가 무시되던 문제가 수정되었다.
- `.fileExporter(_:...)`와 `.fileMover(_:...)`를 같은 view에 적용할 때 일부 dialog가 올바르게 표시되지 않던 문제가 수정되었다.
- `ScrollView`의 non-scrollable axis에서 `containerRelativeFrame(_:alignment:)`가 safe area inset을 잘못 계산하던 문제가 수정되었다.
- SwiftUI type에 대한 retroactive `Equatable` conformance가 SwiftUI value comparison에 사용되지 않던 문제가 수정되었다.
- sheet와 popover에서 `controlSize`, `buttonSizing`, `buttonRepeatBehavior`, `menuIndicatorVisibility`, `ButtonBorderShape` environment value가 default로 reset되지 않던 문제가 수정되었다.
- `List` `Section` header/footer 안에 icon과 title이 모두 있는 `Button`을 둘 때 spacing이 잘못되던 문제가 수정되었다.
- `.navigationTransition(_:)`가 있는 `fullScreenCover` 표시 중 `.onAppear`에서 `@FocusState`를 `true`로 설정할 때 keyboard animation이 zoom transition과 동시에 진행되지 않던 문제가 수정되었다.

## Known Issues

`DocumentReader.read(from:progress:)`와 `DocumentWriter.write(snapshot:to:previous:progress:)`에서 보고한 progress가 표시되지 않을 수 있다.

## 정리

SwiftUI 27의 큰 방향은 다음과 같다.

- 기존 SwiftUI 상태 관리와 builder API를 macro / unified builder 중심으로 정리
- container 수준의 interaction 확장
- toolbar와 tab bar의 적응형 UI 제어 강화
- URL 기반 document workflow 강화
- gesture 입력 source 세분화
