# Xcode 27 Beta Release Notes

> 출처
> - [Xcode 27 Beta Release Notes](https://developer.apple.com/documentation/xcode-release-notes/xcode-27-release-notes)
> - [iOS & iPadOS 27 Beta Release Notes](https://developer.apple.com/documentation/ios-ipados-release-notes/ios-ipados-27-release-notes)
>
> 기준: Xcode 27 beta

## Overview

Xcode 27 beta에는 Swift 6.4와 다음 SDK가 포함된다.

- iOS 27
- iPadOS 27
- tvOS 27
- macOS 27
- visionOS 27

Xcode 27 beta는 다음 OS에서 on-device debugging을 지원한다.

- iOS 17 이상
- tvOS 17 이상
- watchOS 10 이상
- visionOS

Xcode 27 beta는 macOS Tahoe 26.4 이상에서 실행해야 한다.

## Coding Intelligence

### New Features

#### Google Gemini 지원

coding assistant에서 Google Gemini를 사용할 수 있다.

#### Plan mode 강화

agent planning이 Xcode의 first class workflow로 제공된다.

plan은 conversation 옆에 editable Markdown artifact로 표시되며, 사용자는 plan을 검토, 주석, 토론, 승인한 뒤 agent가 진행하도록 할 수 있다.

#### Conversation UI 개편

Coding Assistant sidebar가 conversation 표시와 정리에 집중하도록 변경되었다.

지원 기능은 다음과 같다.

- real-time status
- unread indicator
- drag-and-drop grouping
- archive
- rename
- 여러 conversation bulk group/archive/delete
- 새 tab/window/editor pane에서 conversation 열기

#### Preview Snapshot MCP tool 개선

Preview Snapshot MCP tool이 다음 variant render를 지원한다.

- light/dark appearance
- portrait/landscape orientation
- type size override
- Widget timeline
- Live Activity toggle state

#### Agent의 UI 검증 기능

Xcode agent가 simulator를 boot하고, app을 install/launch하고, touch event를 합성하고, screenshot을 capture해 UI behavior를 검증할 수 있다.

#### String Catalog MCP tool

String Catalog file의 translation을 읽고, 계획하고, 편집하는 MCP tool이 추가되었다.

#### Xcode MCP server 확장

Xcode MCP server가 다음 작업을 위한 tool을 추가했다.

- active run state 조작
- debugger console interaction 및 content 읽기
- scheme 목록 조회 및 switching
- run destination 목록 조회 및 switching
- build settings inspection/modification
- compiler flags inspection/modification
- entitlements inspection/modification
- `Info.plist` key inspection/modification

#### Project insights

agent가 project insight에 접근할 수 있다.

예시는 다음과 같다.

- crash
- disk write
- energy
- hang
- launch issue

#### Built-in specialists

Xcode에는 특정 작업을 위한 Apple-built specialist가 포함된다.

- localization
- UIKit resizing
- accessibility

#### Agent plug-ins

Xcode agent는 skill, MCP server, ACP agent configuration을 포함하는 plugin으로 확장할 수 있다.

skill은 slash command로 실행할 수 있고 completion을 지원한다.

#### Agent filesystem security

Coding Intelligence 설정에서 coding agent와 agent가 spawn한 process의 filesystem access를 monitor/control하는 security layer를 활성화할 수 있다.

#### Agent Client Protocol

Xcode가 Agent Client Protocol을 지원한다.

### Known Issues

- plan-mode confirmation bar가 agent response streaming 중 표시될 때 Yes/No를 누르면 in-flight turn 위에 새 agent turn이 생겨 conversation state가 꼬일 수 있다. agent 응답이 끝난 뒤 확인해야 한다.
- agent plugin에 추가된 ACP agent가 Xcode relaunch 전까지 UI에서 사라지지 않을 수 있다.
- 이전 Xcode가 active developer tool로 설정된 시스템에서 beta deep link를 열면 이전 설치본이 link를 가져갈 수 있다. `xcode-select`로 Xcode 27 beta를 active selection으로 설정해야 한다.
- Apple-authored agent skill이 Codex에서 보이지 않을 수 있다. workaround는 Xcode 27을 default Xcode로 설정한 뒤 `xcrun agent skills export --replace-existing ~/Library/Developer/Xcode/CodingAssistant/codex/skills/__xcode`를 실행하는 것이다.

## Debugging

### New Features

- bridging header를 사용하는 project에서 LLDB가 DerivedData의 explicitly built Swift module과 PCH를 직접 import할 수 있다. bridging header가 있는 debug session의 첫 expression 또는 `po` 속도를 크게 개선할 수 있다.
- LLDB에 `language swift task tree` command가 추가되었다. debugger가 알고 있는 Swift Task tree를 출력한다.
- LLDB가 standard library와 system framework의 `~Copyable` field를 가진 data type을 inspect할 수 있다.
- LLDB가 MCP server인 `lldb-mcp`를 포함한다.

## Interface Builder

### New Features

UIKit, Cocoa Touch 기반 Interface Builder document에 새 compilation mode인 `toolchain`이 도입되었다.

`toolchain` mode는 기본 활성화되며, simulator를 다운로드하지 않아도 IB document를 compile할 수 있다.

build server에서 유용하다.

문제가 있으면 다음 설정으로 이전 방식에 opt out할 수 있다.

```text
IBC_COCOATOUCH_COMPILER_MODE = simulator
```

또는 `ibtool` 직접 호출 시 다음 option을 사용할 수 있다.

```bash
--cocoatouch-compiler-mode simulator
```

## Localization

### New Features

- localization export가 implementation file뿐 아니라 header file의 `NSLocalizedString` 및 유사 macro도 추출한다.
- Xcode agent로 String Catalog string을 번역할 수 있다.
- Xcode는 project setting에 language를 추가하고, 누락된 String Catalog를 만들고, agent가 번역할 수 있도록 context를 제공한다.
- localization comment가 `do not translate`인 string은 String Catalog에서 "Don't Translate"로 자동 표시되고 exported XLIFF에서는 `translate="no"`로 표시된다.
- machine translation으로 번역된 exported XLIFF는 `state-qualifier="leveraged-mt"`를 사용한다.
- String Catalog editor에 Generate Translations button이 추가되었다.

### Resolved Issues

- table parameter를 받는 AppIntent API로 선언된 automatic string이 String Catalog에서 제거될 수 있던 문제가 수정되었다.
- target에 여러 Asset Catalog가 있고 `BUILD_ONLY_KNOWN_LOCALIZATIONS`를 사용할 때 crash가 발생할 수 있던 문제가 수정되었다.

## Previews

### New Features

- Canvas가 새 `#Preview(arguments:)` syntax에 전달된 argument별 preview grid를 표시할 수 있다.
- argument 또는 variant grid의 preview를 누르면 Interactive mode로 열린다.
- Command key를 누른 상태에서 zoom/scroll input event를 canvas가 처리할 수 있다.
- Editor > Canvas menu의 `Send Command-Modified Input to Canvas` toggle로 위 동작을 끌 수 있다.
- iOS preview에 Resizable Canvas mode가 추가되어 임의 크기 container에서 preview를 볼 수 있다.
- `#Preview`와 `#Playground` tab을 canvas에서 각각 pin할 수 있다.

### Known Issues

- 설치되지 않은 runtime으로 preview를 시도하면 placeholder view 대신 다른 preview destination으로 fallback될 수 있다.
- macOS의 Run as App preview 기능 사용 중 잘못된 error banner가 표시될 수 있다.

## Simulator

### New Features

`simctl`과 `devicectl`이 `reboot` command로 simulator reboot를 지원한다.

### Known Issues

- visionOS simulator view 아래 Device Hub toolbar에 keyboard capture mode control이 추가로 표시될 수 있다.
- visionOS simulator view 아래 Device Hub toolbar에 Simulate Trackpad or Mouse mode control이 추가로 표시될 수 있다.
- 실행 중인 visionOS simulator를 reboot한 뒤 interaction이 불가능해질 수 있다. workaround는 Device Hub application을 종료 후 재시작하는 것이다.
- simulator가 delete 후에도 완전히 clear되지 않을 수 있다. workaround는 machine 재시작 후 다시 제거하는 것이다.

## Source Editor

새 Markdown editor가 추가되었다.

project와 agent가 생성한 Markdown file을 rendered form으로 볼 수 있고, formatting tool로 Markdown을 생성/수정할 수 있다.

source code view와 rendered preview는 Xcode canvas에서 사용할 수 있다.

## Swift Compiler

Swift dependency scanner가 single dependency-scan action 동안 Clang module lookup에서 중복 setup work와 header search를 줄이도록 최적화되었다.

이 변경으로 scanning performance가 개선되지만, 하나의 Swift dependency-scan action에서 접근 가능한 모든 Clang module은 고유한 module name을 가져야 한다.

같은 이름의 Clang module을 선언한 module map이 여러 개 보이면 scan error가 발생할 수 있다.

주의해야 할 경우는 다음과 같다.

- project나 SDK가 같은 Clang module name을 여러 header search path 위치에서 제공하는 경우
- vendored third-party source가 SDK module을 다시 선언하는 `module.modulemap`을 포함하는 경우

## Swift Package Manager

### New Features

- package의 test target에서 test failure가 발생하면 `swift test`가 마지막에 요약을 표시한다.
- `swift test`가 test case 반복 실행을 지원한다.

```bash
swift test --maximum-repetitions 10 --repeat-until fail
```

`--repeat-until`에는 `pass` 또는 `fail`을 지정할 수 있다.

## Testing

### New Features

- Test Plan에서 UI testing 중 target application crash에 대한 severity level을 설정할 수 있다. 값은 off, warning, failure, fatal failure이며 default는 failure다.
- Test Plan configuration tab에 filter가 추가되었다.
- Test Navigator에 recent tests와 open tests filter가 추가되었다.
- XCTest assertion을 Swift Testing test 안에서 호출하거나 Swift Testing assertion을 XCTest test 안에서 호출했을 때 assertion failure가 warning severity runtime issue로 표시된다.
- 이 동작은 test plan의 Swift Testing and XCTest Interoperability setting으로 제어한다.
- test plan JSON이 tag를 정렬하도록 업데이트되어 diff review가 쉬워졌다.

### Resolved Issues

- `XCTestObservation` callback인 `testCaseWillStart(_:)`, `testCaseDidFinish(_:)`에서 `XCTestCase.add(_:)`로 attachment를 추가했을 때 result에 persist되지 않던 문제가 수정되었다.
- `Test Repetition Mode`가 entire test plan이 아니라 개별 Swift Testing test case만 반복하도록 수정되었다.
- Swift Testing이 detached Swift task, dispatch queue, background thread에서 발생한 issue를 생성한 test와 더 잘 연결한다.
- parameterized test case가 많은 큰 Swift Testing suite의 성능이 개선되었다.

### Known Issues

- swift CLI로 test를 실행할 때 Swift Package project의 Swift Testing test에서 XCTest cross-framework issue가 표시되지 않을 수 있다. Xcode에서 test를 실행하는 것이 workaround다.
- watchOS Unit/UI test가 device에서 실행되지 않을 수 있다. simulator 실행이 workaround다.

## StoreKit Testing in Xcode

### New Features

- In-App Purchase offer code 설정을 위한 새 StoreKit configuration UI가 추가되었다.
- Transaction Manager에서 IAP offer code로 purchase를 test하는 off-device purchase option이 추가되었다.
- StoreKit Configuration에서 Subscription Bundle local testing을 설정할 수 있다.
- StoreKit Configuration에서 Subscription Suite local testing을 설정할 수 있다.
- Xcode Transaction Manager를 통해 1개월 및 1년 auto-renewing subscription의 volume purchase transaction 생성이 지원된다.
